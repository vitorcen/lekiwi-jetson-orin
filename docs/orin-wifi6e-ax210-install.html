Orin 上启用 Intel AX210 Wi-Fi 6E Bringing up the Intel AX210 Wi-Fi 6E on Jetson Orin (L4T R36.5)

目标板：Orin Nano 8GB ｜ 网卡：Intel AX210 160MHz（PCIe，2725:0024 rev 0x420，RF GF）｜ L4T R36.5.0 / 内核 5.15.185-tegra ｜ 日期：2026-07-18

TL;DR

  • 硬件在、固件也在（/lib/firmware/iwlwifi-ty-a0-gf-a0-*.ucode），唯独 Tegra 内核没编 iwlwifi 驱动模块 —— 所以设置里没有 Wireless。The firmware ships; the kernel module doesn't.
  • 解法：装 Intel 官方 backport-iwlwifi-dkms，DKMS 对着已装的内核头文件现编 iwlwifi/iwlmvm/mac80211/cfg80211 一整套。不用重编内核。
  • 唯一的坑：modprobe iwlmvm 报 Invalid argument —— 开机时常驻的内核自带 cfg80211 与 backport 的符号版本对不上。modprobe -r cfg80211 卸掉再加载即可（免重启）。
  • 成功后网卡名是 wlP1p1s0（predictable naming，不是 wlan0），nmcli 立刻能扫网。

1. 症状与诊断Symptom & triage

GNOME 设置里没有 Wireless、ip link 里没有无线网卡。先分层确认「硬件 / 固件 / 驱动」哪一层缺：

# 硬件在吗？ → 看到 AX210 = 在
lspci | grep -i network
# → Network controller: Intel Corporation Wi-Fi 6 AX210/AX211/AX411 160MHz

# 有无线网卡接口吗？ → 没有任何 wl* = 驱动没起来
ip -o link show
# → lo / enP8p1s0(有线) / can0 / usb0 …  无 wl*

# 驱动模块在内核树里吗？ → 关键判据
modinfo iwlwifi
# → 报错找不到 = iwlwifi 根本没编进这个 Tegra 内核

# 固件在吗？ → AX210 需要 ty-a0-gf-a0-*.ucode
ls /lib/firmware/iwlwifi-ty-a0-gf-a0-*
# → 一堆版本都在，固件不缺

# 无线模块目录里有 intel 吗？
ls /lib/modules/$(uname -r)/kernel/drivers/net/wireless/
# → ath broadcom marvell ti …  就是没有 intel/iwlwifi
✓ 诊断结论 — what's actually missing 硬件 ✅、固件 ✅、cfg80211 配置层 ✅，只差 iwlwifi/iwlmvm 内核模块。L4T 的 5.15.185-tegra 默认 CONFIG_IWLWIFI 没开——这在 Jetson 上很常见，不是你装漏了固件。

2. 模块栈：装的到底是什么The module stack DKMS builds

3. 安装：backport-iwlwifi DKMSInstall

前提：内核头文件已装（ls /usr/src/linux-headers-$(uname -r)* 有目录，L4T BSP 一般自带），DKMS 才能现编。

sudo apt update
sudo apt install -y backport-iwlwifi-dkms   # DKMS 自动对 tegra 头文件编译四个模块

# 编译要几分钟，结束后确认已 installed（不是只 added）
dkms status | grep iwl
# → backport-iwlwifi/11510, 5.15.185-tegra, aarch64: installed
⚠ 别急着判失败 — the build is slow, not stuck apt install 期间 DKMS 会同步编译四个模块（make.log 里几百个 CC），要几分钟。中途看 dkms status 显示 added 是正在编，编完才变 installed。别中断 apt。

4. 唯一的坑：cfg80211 符号版本冲突The one gotcha

装完 modprobe iwlmvm，撞上：

modprobe: ERROR: could not insert 'iwlmvm': Invalid argument

根因：开机时内核自带的 cfg80211 已经常驻内存（引用计数 0，没人用，但占着位），而 backport 的 iwlmvm 期望的是 backport 自带 cfg80211/mac80211 的符号版本（modversions CRC）。两套 CRC 对不上 → 内核拒绝插入 → EINVAL。Not a build error — a stale in-tree cfg80211 already resident with different symbol CRCs.

# 卸掉常驻的旧 cfg80211（refcount 0，安全），再成套加载 backport
sudo modprobe -r cfg80211
sudo modprobe iwlmvm            # 自动带出 backport 的 cfg80211→mac80211→iwlwifi→iwlmvm
✗ 若 modprobe -r cfg80211 说 in use — fall back to reboot 说明有别的东西正用着它，那就 sudo reboot。重启后 depmod 已把 backport 模块放进优先级更高的 updates/dkms/，开机会成套加载正确版本，一样出 wl*。

5. 验证Verify

# 内核日志：固件加载 + 识别 AX210
sudo dmesg | grep -i iwlwifi | tail
# → loaded firmware version 86.… ty-a0-gf-a0-86.ucode op_mode iwlmvm
# → Detected Intel(R) Wi-Fi 6 AX210 160MHz, REV=0x420
#   （WRT: Invalid time point / Failed to allocate TLV 是 AX210 已知无害告警，忽略）

# 四个模块成套加载
lsmod | grep -E 'iwlmvm|iwlwifi|mac80211|cfg80211'

# 网卡出现（注意名字是 wlP1p1s0，不是 wlan0）
ip -o link show | grep -i wl

# 能扫到网就成了
nmcli device wifi list | head
✓ 成功判据 — how to know it worked ① lsmod 见到 iwlmvm/iwlwifi/mac80211/cfg80211 四件套；② ip link 出现 wlP1p1s0；③ nmcli device wifi list 能列出周边 SSID；④ GNOME 设置里出现 Wireless。连网只需 nmcli device wifi connect <SSID> password <密码>（密码不入库）。

6. 持久化与后续Persistence & caveats

  • 重启后自动加载：DKMS 模块在 updates/dkms/（优先级高于内核自带），本次要手动卸 cfg80211 只是因为它在装 DKMS 之前就已常驻。建议装完重启一次做回归验证，确认开机自动出 wl*（会短暂打断 base_host / pad_teleop）。
  • 内核升级：只要 linux-headers 在，DKMS 会对新内核自动重编；若升级后无线消失，先 dkms status 看是否 installed，需要时 sudo dkms autoinstall。
  • 蓝牙：AX210 的蓝牙（8087:0032）走 btintel，本就正常（Xbox 手柄即经它连接），与本文的 WiFi 驱动无关。
