ROS 2 Humble 安装记录 Installing ROS 2 Humble Desktop on Jetson Orin Nano Super (JetPack 6.2.2)

目标：Orin Nano 8GB @ 192.168.13.189 ｜ Ubuntu 22.04 / L4T 36.5 ｜ ROS 2 humble 0.10.0 ｜ 日期：2026-07-16

TL;DR

  • Ubuntu 22.04 → 官方对应 ROS 2 Humble（Tier-1，支持到 2027），装 ros-humble-desktop（带 RViz2）。
  • 国内网络必须换源：Ubuntu ports.ubuntu.com 从国内连不上 → 换清华 TUNA 镜像；ROS 源同样走 TUNA。
  • 装 ROS 前先做系统安全更新（新刷板子落后半年补丁），但 apt-mark hold 锁死 nvidia-l4t 包，别让升级动了刷好的 bootloader/内核。
  • NVIDIA 源（repo.download.nvidia.cn）提供 CUDA 加速的 libopencv-dev，ROS 会自动用上。
  • 验收标准不是"装没装"，而是 talker/listener 能真通信 + ros2 node/topic 能内省。

3. 完整步骤Step by step

① 修网络：换国内镜像 （不做这步，apt 直接超时）

❌ 坑 1：国内连不上 ports.ubuntu.com — Ubuntu ARM mirror unreachable from CN Orin 用的是 arm64，Ubuntu ARM 包在 ports.ubuntu.com（非 x86 的 archive.ubuntu.com）。该域名国内 IPv4 超时、IPv6 直接 Network unreachable。板子能 ping 通 8.8.8.8（说明有外网），单单这个源连不上。 
# 只改 ports.ubuntu.com，保留 nvidia-l4t 源不动，留备份
sudo cp -n /etc/apt/sources.list /etc/apt/sources.list.bak
sudo sed -i 's|http://ports.ubuntu.com/ubuntu-ports|https://mirrors.tuna.tsinghua.edu.cn/ubuntu-ports|g' \
     /etc/apt/sources.list

# 强制 apt 走 IPv4（板子 IPv6 不通）
echo 'Acquire::ForceIPv4 "true";' | sudo tee /etc/apt/apt.conf.d/99force-ipv4

sudo apt-get update      # TUNA ~5 MB/s，秒级完成

备选镜像（都实测可达 443）：mirrors.ustc.edu.cn、mirrors.aliyun.com、repo.huaweicloud.com。

② 系统安全更新（装 ROS 前的卫生）

⚠ 坑 2：升级前先 hold 住 nvidia-l4t 包 — protect the flashed bootloader/kernel 新刷的板子内核/bootloader 来自 NVIDIA 源。虽然本次 334 个可升级项里不含 nvidia 包，仍先 apt-mark hold 上保险，防止依赖链意外拉动它们。 
# 锁死所有 nvidia-l4t / tegra / jetpack 包
sudo apt-mark hold $(dpkg -l | awk '/nvidia-l4t|nvidia-tegra|nvidia-jetpack/{print $2}')

# 应用更新（noninteractive + 保留旧配置，避免交互卡住 SSH）
sudo DEBIAN_FRONTEND=noninteractive apt-get -y \
     -o Dpkg::Options::=--force-confold upgrade

sudo reboot              # glibc/systemd 更新，重启生效

本次结果：334 → 2 可升级（剩 2 个是 phased 灰度更新），302 个安全补丁应用，ca-certificates 更新（新导入 16 张根证书）。network-manager.service not found 报错无害 —— 板子用 systemd-networkd，本就没装 NetworkManager。

③ 加 ROS 2 源

❌ 坑 3：echo ... | sudo -S tee file 会写空文件 — stdin collision 用"密码从 stdin 喂 sudo"的写法时，echo "内容" | sudo -S tee 文件 里 sudo -S 把 stdin 当密码读走了，管道进来的内容全丢，源文件写成空的、apt 看不到 ROS 源。
正解：内容先写到用户可写的 /tmp，再 sudo cp 到位，避开 stdin 冲突。 
# GPG key（dearmor 成二进制 keyring）
curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /tmp/ros.key
sudo gpg --dearmor -o /usr/share/keyrings/ros-archive-keyring.gpg /tmp/ros.key

# 源文件：先写 /tmp 再 cp（走 TUNA 的 ros2 镜像）
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] \
https://mirrors.tuna.tsinghua.edu.cn/ros2/ubuntu jammy main" > /tmp/ros2.list
sudo cp /tmp/ros2.list /etc/apt/sources.list.d/ros2.list
sudo apt-get update

④ 安装 ROS 2 Humble desktop

sudo DEBIAN_FRONTEND=noninteractive apt-get install -y \
     ros-humble-desktop \
     ros-dev-tools \
     python3-colcon-common-extensions \
     python3-rosdep

# rosdep 初始化
sudo rosdep init
rosdep update

# 环境自动加载
cat >> ~/.bashrc <<'EOF'

# ROS 2 Humble
source /opt/ros/humble/setup.bash
export ROS_DOMAIN_ID=0
EOF
✓ 加成：CUDA 加速的 OpenCV — comes from the NVIDIA repo, not Ubuntu 安装日志里 libopencv-dev 从 repo.download.nvidia.cn/jetson r36.5 拉取，而非 Ubuntu 源 —— ROS 视觉相关包会自动用上 Jetson 的硬件加速 OpenCV。占用 1.67 GB，装后根分区 12G / 227G（6%）。

⑤ 验证（真通信，不只是装上）

source /opt/ros/humble/setup.bash

# 终端 A：发布者
ros2 run demo_nodes_cpp talker
# 终端 B：订阅者 —— 应打印 "I heard: [Hello World: N]"
ros2 run demo_nodes_py listener

# 内省
ros2 node list      # /talker  /listener
ros2 topic list     # /chatter /rosout /parameter_events
ros2 topic hz /chatter
✓ 本次验收结果 talker 发布 7 条、listener 收到 5 条（差值 = listener 晚 2 秒启动，正常）；/chatter 频率 ~1 Hz；ROS_DISTRO=humble、rmw_fastrtps_cpp、283 个包在 /opt/ros/humble/share。DDS pub/sub 全通。
⚠ 坑 4：别用 pkill listener 清 ROS 节点 — it kills sshd sshd 的监听进程在 ps 里显示为 sshd: ... [listener]，pkill -f listener / pkill listener 会误杀它，直接断掉你自己的 SSH（exit 255）。清 demo 节点请用精确名：pkill -f demo_nodes_cpp。另外 ros2 run 会 fork 子进程，kill 父进程可能漏掉节点子进程，也用 pkill -f 精确匹配节点可执行名。
