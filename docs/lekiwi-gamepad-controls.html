LeKiwi 手柄键位说明Gamepad control map — board-side pad_teleop daemon

通用 USB 手柄(DragonRise 0079:181c)接 Orin，板上 pad_teleop.py 用 evdev 读、转 ZMQ 推给 base_host。开机自启，无需 GUI。

TL;DR

  • 右手管底盘：右摇杆前后走 / 左右转，十字键数字满速开车。Right stick + D-pad = drive base.
  • 左手管机械臂：左摇杆前后 = 伸展/收回(2+3号)、左右 = 底座平移(1号)；Y/A = 4号腕升降；X/B = 腕旋转(5号)；右肩/右扳机 = 爪子开合(6号)。
  • 安全：SELECT 按住 = 瞬时急停(松开即恢复)；START = 收臂回休息位并断扭矩(松弛)。
  • 对码：GUI 底部日志会实时打出你按下的真实事件码 —— 键位不对照此改 BTN_LABELS 即可。

系统与安全 System & safety

手柄事件码动作
LB(左肩)BTN_TL底盘 降速档
LT(左扳机)BTN_TL2底盘 升速档
SELECT 按住BTN_SELECT瞬时急停：一切归零(松开即恢复，无闩锁)
STARTBTN_START收臂回休息位 → 断扭矩(松弛，可手摆)；碰任意臂输入即唤醒
为什么急停做成"按住"而不是"一按锁定"？ 杂牌手柄丝印 ≠ 事件码：一按锁定的急停，解锁键往往按不对，车看着像坏了。急停一律做成瞬时(momentary)，松手即恢复。

对码：GUI 底部日志 Remap via the GUI log

手柄接在板子上、GUI 在桌面，两者原本互不可见。为此加了一条单向日志总线：pad_teleop.py 每次按键 / 推杆都往 tcp://*:5556(PUB)广播一行，GUI 底部日志栏实时显示。

手柄  BTN_TR ↓  爪张开
手柄  ABS_Y +0.72  臂前伸/收
手柄  ABS_HAT0X -1.00  底盘转向
键位不对怎么办 按一下那个键，看底部日志打出的真实 BTN_xxx / ABS_xxx，去 board/home/jatson/pad_teleop.py 的 BTN_LABELS / 映射处改，跑 scripts/deploy_board.sh 重新部署即可。无需 ssh 翻 journal。

部署 Deploy

# 首次一次性：ssh 免密 + 装单元 + NOPASSWD 规则（唯一要 sudo 的一次）
ssh-copy-id jatson@192.168.3.189        # ssh/scp/rsync 从此免密，密码不入库
scripts/setup_board.sh 192.168.3.189    # 只跑一次

# 之后改完 board/ 里的程序，一条命令 rsync + 重启，全程免密
scripts/deploy_board.sh 192.168.3.189

# 看日志
ssh jatson@192.168.3.189 'journalctl -u pad_teleop -u base_host -n 30 --no-pager'
