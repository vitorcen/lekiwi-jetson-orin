LeKiwi ROS 2 集成计划

日期：2026-07-20  |  状态：PLAN 经 codex gpt-5.6-sol 评审修订  |  触发：USB 深度相机已接（型号待上电确认），单线激光雷达在途

⏵ 一句话方案

ROS 2 作为并行能力层接入，不替换现有 ZMQ 栈：base_host 仍是串口唯一属主 + 唯一控制仲裁器，ROS 只是第 4 个控制源（/cmd_vel→ZMQ 5555）；安全门（雷达净空限速）下沉到 base_host 仲裁之后、电机之前，对所有来源生效。导航栈走最保守的成熟组合： slam_toolbox 建图 → AMCL 定位 → Nav2 + DWB 先跑通，MPPI 后验 A/B。里程计可信之前不碰 SLAM。
决策点选择理由
ROS vs 现有 ZMQ 栈并行共存，base_host 不动摇生产链路（手柄/GUI/语音控车）零破坏；串口只能有一个属主
控制优先级pad > gui > ros_nav > mcp人工现场 > 人工远程 > 本地自主 > 云端 LLM（延迟与不确定性递增）
安全急刹位置base_host 内、仲裁后放 ROS 侧只拦 Nav2 = 手柄/GUI/MCP 全绕过，那不叫安全层
建图 / 定位slam_toolbox（离线建图）+ AMCL（生产定位）AMCL 不是 SLAM，本就是两阶段；Cartographer 双套维护 砍
ControllerDWB 先跑通 → MPPI A/BDWB 配好 vel_y 也能全向；TEB Humble 维护状态差 砍
深度相机一期只做预览 + rosbag，不进 costmap参考项目实证：避障 100% 靠单线雷达就够了；二期再 depthimage_to_laserscan
Isaac ROS / nvblox不碰二维室内导航 + 8GB 已常驻 VLM，引入 CUDA 管线纯属找麻烦
rosbag vs lerobot 数据集两条线独立，共享 run_uuid + 时间基准硬统一格式是假需求，只会造一个没人敢改的中间格式

参考项目的反面教训（同样值钱） 
1. 深度相机(Astra Pro)从未进 costmap，只做 GUI 预览——避障 100% 靠单线雷达，够用。
2. Astra 厂商 ROS 驱动会关 IR 激光致深度恒 0，最后弃用改裸 OpenNI2 直读——驱动选型要留 PlanB。
3. USB2 带宽：深度(bulk)+彩色(UVC iso)同开互掐，深度掉到 1.5fps——lsusb -t 看根节点分布。
4. DWB 没吃透全向横移（当近似差速用），想换 TEB 未遂——我们直接把 vel_y 配对，后验 MPPI。
5. 单线雷达(MS200)在 ~0.1m 死区和吸光表面报 0.0——避障逻辑把「扇区无有效回波」判堵不判空。
6. nav2 真实参数文件只在板上没进仓库——我们的参数文件必须进 repo（board/ 镜像范式已有）。
7. 横移故障（vy 指令下原地旋转，后轮扭矩弱）——全向底盘先标定解耦再谈导航。
8. ros2 topic pub 喂 /initialpose 必须 -w 1 等订阅者，否则丢包；nav2 停止必超时，重启前 reset-failed。

3. 目标架构

核心数据结构只有一条：多控制源 → 唯一仲裁器 → 最终安全门 → 电机。这个链只能有一套，全在 base_host 里。 ROS 侧不做第二个 mux、不做第二个 watchdog。

3.1 控制注入协议升级 必改

  • 命令帧增加 src + seq + 单调时间戳 + TTL，未知字段旧客户端可忽略（零破坏）。
  • base_host 每周期排空队列、按源取最新且未过期的命令——杜绝 ZMQ 积压导致旧速度复活（watchdog 只看「最近收到」时会把积压当新鲜）。
  • 控制租约与零速流分离：Nav2 idle 时 bridge 不许以 10Hz 零速流永久占住 ros 档；零速≠活跃。
  • 角速度单位换算收口在 bridge：ROS rad/s → base_host deg/s。

3.2 安全门语义 必改

  • lekiwi_safety（ROS 侧，搬 yahboom 算法）从 /scan 算带有效期的方向净空限速/停止裁决，PUB :5558；base_host 对仲裁后的最终命令裁剪——手柄/GUI/Nav2/MCP 谁都绕不过。
  • 模式区分：自主模式（ros/mcp 档在驱动）裁决过期→停；手柄模式→显式配置为低速爬行或停，不许偷偷 fail-open。
  • watchdog（命令源死亡）与雷达急刹（前方有障碍）是两回事，都保留。
  • 每次限速/停车记录原因、最近扫描时间、触发扇区（进 5556 日志总线 + /diagnostics）。
  • 软件急刹不冒充硬件安全：物理急停（断舵机电）待补，列入硬件采购。

3.3 里程计链路 必改

  • base_host 串口循环只批量读轮 7/8/9 的位置/速度（臂 6 电机降频或按需读，不许拖垮底盘闭环），采样即打板端单调时钟，PUB :5557 原始数据 + 通信错误计数。
  • ROS 侧 lekiwi_odom：STS3215 是 12 位/圈 4096 步绝对编码器，多圈掉电不存——软件展开 int64 累计计数（取模差值选满足最大轮速约束的最近增量），检测舵机重启/跳变则丢样本重置基准；开机位置即里程零点。方向符号/每圈计数/轮径/三轮安装角全参数化。
  • 不加 EKF：无第二观测源时 EKF 只是给轮里程计化妆。实测偏航漂移大→先加独立 IMU，再谈融合。

3.4 仓库布局

ros2/                          # colcon 工作区源（deploy_board.sh rsync → 板上 ~/ros2_ws/src）
├── lekiwi_base_bridge/        # /cmd_vel→ZMQ 5555；:5557→/odom + odom→base_footprint TF
├── lekiwi_description/        # 最小 URDF：base_footprint / base_link / 3×wheel / laser / camera optical
├── lekiwi_safety/             # /scan→净空裁决 PUB :5558（rclcpp，搬 yahboom safety_stop 算法）
├── lekiwi_recorder/           # rosbag2 episode 录制（搬 episode_recorder.py）
└── lekiwi_bringup/            # launch + nav2/slam 参数 yaml（参数必须进仓库！）+ systemd units
board/etc/systemd/…            # ros-bridge / lidar / mapping⟂nav2(Conflicts=) 服务，沿用镜像范式
scripts/replay.sh              # rosbag→RViz 回放（独立 ROS_DOMAIN_ID + sim_time）

URDF 必须有 base_footprint（不只 base_link），否则 Nav2 与二维传感器配置后患无穷。 systemd 启动顺序：base_host → bridge/TF → sensors → safety → localization → Nav2。

4. 阶段计划（每阶段带验收门，不过门不进下一阶段）

阶段内容验收门
P0 硬件与总线确认
板子上电即做		 lsusb 认深度相机型号定驱动；lsusb -t 看 USB 根节点分布（深度相机/UVC/雷达/舵机总线是否挤同一 USB2 根）；udev 稳定命名；雷达到货认型号 两相机+雷达+舵机总线同时工作无掉帧断连；/dev/lekiwi_* 稳定命名生效
P1 最小控制桥 协议升级（src/seq/ts/TTL + latest-only 排空，§3.1）；base_host 加 ros 档；lekiwi_base_bridge 订 /cmd_vel 注入 teleop_twist_keyboard 开车；pad 随时抢占；ros 档 idle 不占租约；旧 GUI/pad/MCP 客户端零改动仍工作
P2 里程计与底盘标定
杀错误路线的关卡		 轮反馈 :5557 + lekiwi_odom（§3.3）；最小 URDF + TF；vx/vy/wz 解耦标定（搬 strafe_test 方法论） 轮反馈 ≥30Hz（目标 50）、p95 间隔 <40ms、无 >100ms 空洞、动臂时底盘频率不坍塌；直行 2m 回读误差、原地转 360° 漂移、横移解耦达标。不过门就停，不准进 SLAM
P3 雷达 + 基础录制 雷达驱动→/scan、静态 TF、QoS；lekiwi_recorder（rosbag 示教录制）+ replay.sh；深度相机仅预览+录制；/diagnostics 上线 RViz 里 scan 稳定 10Hz 对齐 TF；录制→回放→RViz 复现全链路通；断电重启后服务自愈
P4 全局安全门 lekiwi_safety :5558 + base_host 裁剪（§3.2） 手推车冲障碍：pad/GUI/ROS/MCP 四源逐一验证被限速/刹停；雷达拔线→自主模式停车、手柄模式按配置降速；日志能答「为什么刹了」
P5 建图与定位 slam_toolbox（async）建图→存图；AMCL 定位（宽协方差 /initialpose，-w 1）；mapping⟂nav2 systemd 互斥 房间地图闭环不撕裂；原地旋转/横移后定位不丢；重启后重定位收敛
P6 Nav2 NavFn + DWB（配好 min/max_vel_y 用上横移）跑通端到端；再 Qwen 满载条件下 A/B MPPI（controller 15-20Hz、p95 计算时间 <50% 周期、RSS/温度/降频） 点到点导航成功率、贴桌腿不撞（inflation 从 0.35 起调）；MPPI 数据不达标就留 DWB
P7 GUI 集成 Tauri ROS Tab 走 rosbridge websocket：节点/话题状态、rosbag 开关、地图显示 + 导航点下发。白名单限制可调用 topic/service，不裸露整个 ROS 图 GUI 下发导航点→车到点；录制开关回显状态；断连自动重连
每次只调一层：不许同时调 SLAM、里程计和 controller，否则不知道哪层坏了。

6. rosbag 与 lerobot 数据集的关系

  • lerobot recorder 仍是模仿学习数据的唯一权威写入方；rosbag 管 ROS 调试、传感器回放、导航复现、标定。
  • 两边共享 run_uuid、开始时间、机器人配置版本、操作者标签；base_host 统一遥测两边都能消费。
  • 不做统一格式：相机编码、动作频率、episode 边界、训练 schema 都不同，硬统一 = 造没人敢改的中间格式。
