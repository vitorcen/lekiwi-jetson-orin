LeKiwi 移植到 Orin Nano：手柄操控底盘与机械臂 Porting LeKiwi to Jetson Orin Nano — gamepad teleop for the mobile base & SO-101 arm

2026-07-16 · 依据维特智能（WitMotion）语雀教程 4 篇 + 本项目已完成的舵机总线 bring-up 整理 · 板子 jatson@192.168.3.189

TL;DR

  • 机器人 = LeKiwi：3 轮全向底盘（3 个 STS3215-C018，ID 7/8/9）+ SO-101 从臂（6 个 STS3215-C018，ID 1~6），9 个电机挂同一条总线、同一块舵机驱动板、一个 USB 口。Leader 主臂另配一块驱动板（本方案手柄替代主臂，Leader 可先不接）。
  • 教程原方案是「树莓派/RDK X5 跑 lekiwi_host + PC 跑遥操作客户端」。移植到 Orin：host 与客户端全部跑在 Orin 上（remote_ip=127.0.0.1），手柄直接插 Orin，少一台机器、少一跳网络。
  • 软件走 lerobot（教程锚定 commit 26ff40d），不引入 ROS —— 遥操作用不上，别过度设计。
  • 实施四步：① 接满 9 电机并扫总线确认 ID → ② Orin 装 miniconda + lerobot → ③ 标定 + 键盘遥操作打通 → ④ 手柄客户端（pygame，~200 行）。
  • 舵机串口这条路本项目已趟平：CH340 驱动（自编 ch341.ko）、brltty 冲突、dialout 权限全部解决，裸协议扫描脚本 ~/scan_servos.py 可直接复用。

1. 硬件拓扑Hardware topology

关键事实（来自《lekiwi 组装》）：

  • 一条总线管全部 9 个电机。从臂 1 号舵机有两个接线口：一口向上串到 2~6 号，另一口的线穿过底盘顶板，插进底盘舵机驱动板的第二个串口座。臂 6 个 + 轮 3 个共享一个 USB 串口。
  • ID 分配：臂 1~6（1 = 最下面的底座关节，6 = 夹爪），轮子 7~9。出厂成品已编好号、已标定（客服提供标定 JSON），自装件才需要 lerobot-setup-motors（编号顺序：臂 6→5→4→3→2→1，再轮 9→8→7，且编号时舵机两两断开）。
  • Leader 主臂是 7.4V 舵机（C044/C001/C046，减速比小便于拖动），单独一块驱动板一条 USB。本方案用手柄替代主臂做遥操作，Leader 可完全不接；以后要示教录数据再上。

第 4 步 · 手柄客户端（本方案核心增量）

手柄插 Orin（USB 或 2.4G 接收器，/dev/input/js0；蓝牙手柄也行但先用有线降变量）。写一个 teleop_gamepad.py 替代键盘客户端，结构与官方 example 相同，只是输入从 keyboard 换成 pygame joystick：

手柄输入底盘模式（默认）机械臂模式
左摇杆 ↑↓ / ←→前后 x.vel / 左右平移 y.vel肩转 shoulder_pan / 肩抬 shoulder_lift
右摇杆 ←→ / ↑↓旋转 theta.vel / —肘 elbow_flex / 腕俯仰 wrist_flex
LB / RB降速档 / 升速档腕旋转 wrist_roll ∓
LT / RT—夹爪开 / 合 gripper
Y 键切换 底盘 ⇄ 机械臂 模式
B 键急停：底盘速度清零 + 臂保持当前位

代码骨架（关节用「当前值 + 摇杆增量」的位置控制，底盘用速度控制——与 LeKiwi 动作空间一致：臂是 *.pos，底盘是 x.vel/y.vel/theta.vel）：

import pygame
from lerobot.robots.lekiwi.lekiwi_client import LeKiwiClient, LeKiwiClientConfig

JOINTS = ["arm_shoulder_pan", "arm_shoulder_lift", "arm_elbow_flex",
          "arm_wrist_flex", "arm_wrist_roll", "arm_gripper"]
DEAD, HZ, STEP = 0.15, 30, 1.5      # 死区 / 控制频率 / 每拍关节步进(度)

pygame.init(); pygame.joystick.init()
js = pygame.joystick.Joystick(0); js.init()

robot = LeKiwiClient(LeKiwiClientConfig(remote_ip="127.0.0.1", id="orin_lekiwi"))
robot.connect()
obs = robot.get_observation()
target = {j: obs[f"{j}.pos"] for j in JOINTS}   # 从当前姿态起步，避免上电跳变

arm_mode, speed = False, 0.4
clock = pygame.time.Clock()
while True:
    pygame.event.pump()
    ax = lambda i: v if abs(v := js.get_axis(i)) > DEAD else 0.0
    action = {f"{j}.pos": target[j] for j in JOINTS}
    action.update({"x.vel": 0.0, "y.vel": 0.0, "theta.vel": 0.0})

    if arm_mode:
        target["arm_shoulder_pan"]  += -ax(0) * STEP
        target["arm_shoulder_lift"] += -ax(1) * STEP
        # ... 其余关节 / LT·RT 夹爪同理，并按标定限位 clamp
    else:
        action["x.vel"]     = -ax(1) * speed        # m/s
        action["y.vel"]     = -ax(0) * speed
        action["theta.vel"] = -ax(3) * 90 * speed   # deg/s

    robot.send_action(action)
    clock.tick(HZ)
✔ 为什么自写而不用 lerobot 自带的 gamepad teleoperator上游的 gamepad 遥操作是给单臂末端（EE）控制设计的，不管底盘；LeKiwi 的键盘 example 又不管手柄。两边各缺一半，而 client 的动作接口（一个 dict）足够简单——直接对着它写 200 行，模式切换、限位、急停全在自己手里，这是最笨也最清晰的做法。

4. 验收标准Acceptance

  • scan_servos.py 扫出 ID 1~9，电压 12V±0.5。
  • 键盘 WASD 能开底盘：前后/平移/旋转方向正确（方向反了是标定或轮序问题，别在手柄层面改符号硬补）。
  • 手柄底盘模式：摇杆推多少走多快，回中即停；B 键急停立即生效。
  • 手柄机械臂模式：6 关节可控、限位内不撞机械结构、松手保持不下坠。

5. 风险与坑Risks & gotchas

① 桌面调试先把轮子架空底盘一通电就是 3 个能跑的轮子。手柄映射没验证前，垫高底盘让轮子悬空，或在空旷地面、手不离 B 键。
② 串口驱动板两种可能维特这块 SERVO-BUS-TTL 板若是 CDC 芯片则出 /dev/ttyACM0（内核原生支持）；若又是 CH340 则出 /dev/ttyUSB0 —— 本项目已自编 ch341.ko 并 depmod 持久化、已卸 brltty，两条路都通。一切以 lerobot-find-port 实测为准。注意 nvidia 内核包升级后 ch341.ko 要重编（apt-mark hold 已挡住意外升级）。
③ Orin 供电别照抄树莓派方案教程的 12V→5V Type-C 降压模块是给树莓派的（5V/3A≈15W），Orin devkit 是 DC 圆口 9~20V，MAXN 下整机可到 25W。上底盘时 Orin 由电池经 DC 分线器直接供电（11.1V 三串锂 9.6~12.6V 恰在范围内），弃用 5V 降压模块；同时留意舵机堵转瞬间的电压跌落，供电线径要够。桌面阶段先用原装适配器。
④ 锚定 commit，别追上游 HEADlerobot API 变动频繁，语雀教程明说「官方文档不保证成功」。全程锁 26ff40d；升级是独立事件，升级后需回归 §4 验收项。
⑤ PyPI 的 torch 在 Jetson 上没有 GPU（已实测踩到） pip install -e ".[lekiwi]" 装进来的是 torch 2.11.0+cu130，而 JetPack 6.2.2 的驱动是 CUDA 12.6 → torch.cuda.is_available() 返回 False（"NVIDIA driver is too old, found 12060"）。 遥操作/标定纯 CPU，本篇不受影响；等到 §6 跑策略推理时必须换 NVIDIA 官方 Jetson torch wheel （JetPack 6.x / CUDA 12），并开 Super 模式。别指望 PyPI 的 torch 能用上 Orin 的 GPU。
⑥ 手柄设备权限/dev/input/js*、event* 通常 input 组可读；若 pygame 找不到手柄，先 jstest /dev/input/js0（apt: joystick 包）确认内核层，再查 usermod -aG input jatson。
